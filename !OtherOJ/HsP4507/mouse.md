# 🐭 Mouse and the Secret Permutation — 扩展版（满分 100 分）

有一只特别的老鼠，它想要找出一个秘密排列  
$$
p[1], p[2], \dots, p[N].
$$

但它自己无法做到，于是它使用了一种特殊的装置——**“排列探测器”**。

---

## 🔍 排列探测器

给定任意一个排列  
$$
q[1], q[2], \dots, q[N],
$$  
这个装置会告诉它：有多少个位置 $i$ 满足  
$$
p[i] = q[i].
$$

你可以调用如下操作：

```
query(q[1..N])
```

- 参数 `q` 必须是一个合法的排列。
- 返回值是与秘密排列相同的位置数。
- 当返回值为 $N$ 时，表示你已经找到正确答案，你必须立即退出 `solve`。

你的目标是：

> **在尽可能少的查询次数内找出整个排列 $p$。**

---

# 🧩 你需要实现

```cpp
void solve(int N)
```

你可以调用：

```cpp
int query(vector<int> q)
```

当你认为你已经知道了 $p$，  
你可以调用一次 `query(p)`，如果返回值为 $N$，说明你答对了，你必须立即结束程序。

---

# 📥 输入格式（由 grader 提供）

本题为函数交互题，**不从标准输入读取任何内容**。

评测程序会：

1. 从标准输入读取三项内容：
   ```
   N mode seed
   ```
2. 使用内置的固定密钥 + `seed` 生成一个随机排列 $p$
3. 调用你的 `solve(N)`
4. 统计你使用的最大查询次数 $Q$
5. 根据子任务评分规则给出得分

评测程序支持 **多组测试**，你需要保证整体效率。

对于非常小的 $N$（如 $4$、$5$），评测程序会自动对所有排列进行穷举查询，以确保你的程序行为正确。

# 📊 数据范围与子任务（总分 100 分）

本题包含 **5 个子任务**，覆盖不同规模与不同强度的测试。

---

## **子任务 1（10 分）—— 极小规模**

- $1 \le N \le 7$
- 允许使用非常多的查询（甚至枚举所有排列）

评分方式：

| 最大查询次数 $Q$ | 得分 |
|------------------|------|
| $Q \le 50$ | 10 |
| $50 < Q \le 200$ | 7 |
| $200 < Q \le 5040$ | 4 |
| $Q > 5040$ | 0 |

---

## **子任务 2（20 分）—— 小规模**

# 🧪 评测说明（grader 行为）

评测程序将：

- 使用固定密钥 + seed 生成随机排列
- 支持多组测试
- 对于小 $N$（如 $4$、$5$），会自动枚举所有排列并调用你的程序，确保正确性
- 使用 SPJ 计算你的得分
- 若你最终 `query(p)` 返回 $N$，则判定为正确
- 若查询次数过多，将显示 “Correct — Too many queries”，得分为 0
