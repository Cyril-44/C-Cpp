# [B - 摆放 / Slaganje](https://www.luogu.com.cn/problem/P15568)
## 题意简述
你有 $n$ 棵一模一样的树，你现在要把它们顶点重合地放到一个 $n$ 边形上，使得每两个顶点之间都有边。
一定有解，需要输出构造的方案。
## 做法
我想不到。
[题解链接](https://www.luogu.com.cn/article/bj1oijdm)
题解猜测，一定先假设一个顶点对应集合 $p$，即树上顶点 $i$ 和多边形顶点 $p_i$ 重合，然后对这个 $p$ 进行循环移位构造可行方案。
那么问题就简化成要求任意距离($1\sim \lfloor \frac{n}{2} \rfloor$)的点对都出现在初始 $p_i$ 中。
思索之后发现菊花是比较好的结构，我们只需要找到一些菊花，它们的总点数大于 $\lfloor \frac{n}{2} \rfloor$ 即可，这是容易的。

# P9330 [JOIST 2023] JOI 国的节日 2 / Festivals in JOI Kingdom 2

## 题目描述

在 JOI 王国，每年都会举行一次全国性的节日。在节日期间，总共有 $N$ 个活动。每个活动的时间表已经固定。$N$ 个活动的时间表由长度为 $N$ 的序列 $a, b$ 描述，满足以下条件：

- 从 $1$ 到 $2N$ 之间的每个整数都出现在 $a$ 或 $b$ 中。
- $a_i < b_i \ (1 \le i \le N)$。
- $a_i < a_{i + 1} \ (1 \le i \le N - 1)$。

第 $i$ 个活动将在节日开始后的 $a_i$ 分钟开始，并在节日开始后的 $b_i$ 分钟结束。

节日的参与者可以选择参加任意活动。然而，不允许参加时间表重叠的两个活动。（注意，活动的开始时间和结束时间彼此不同。）

JOI-kun 想参加尽可能多的活动。直到去年，他通过计算机使用以下程序选择参加的活动：

> 对于 $i = 1, 2, \dots, N$，按以下顺序进行。
> > 如果第 $i$ 个活动的时间表与他已经选择参加的其他活动的时间表不重叠，他将参加第 $i$ 个活动。否则，他将不参加第 $i$ 个活动。

然而，在学习了计算机科学之后，JOI-kun 注意到上述算法并不一定能最大化他参加的活动数量。从今年开始，JOI-kun 将使用改进的算法。使用改进的算法，JOI-kun 将能够最大化他参加的活动数量。

JOI-kun 想知道使用改进算法时，产生更多活动数量的情况数。

编写一个程序，给定整数 $N$ 和一个大质数 $P$，计算出描述 $N$ 个活动时间表的序列 $a, b$ 的对数，其中改进的算法产生更多的活动数量。由于答案可能非常大，程序应输出答案除以 $P$ 的余数。

对于所有测试数据，满足 $1 \le N \le 2 \times 10 ^ 4$，$10 ^ 8 < P < 10 ^ 9$，保证 $P$ 是质数，保证所有输入均为整数。

# [E - Super Backpack](https://qoj.ac/problem/14123/statement/zh_cn)
## 题面描述
有 $n$ 种物品。Vasya 拥有第 $i$ 种物品 $k_i$ 个。第 $i$ 种物品中的第 $j$ 个物品占用 $j$ 个单位的体积，且价值为 $a_{i,j}$。Vasya 想要从每种物品中恰好选择一个，使得它们的总体积恰好为 $t$。请帮助 Vasya 确定他能选择的物品的最大总价值。
## 关键 trick
有四种重量的物品分别为 $1,2,3,4$，如果选择一些物品的总重量为 $24$，那么一定存在这些物品的一个子集重量和为 $12$。
我不会证明，而且猜测如果物品重量为 $1\sim n$ 如果选择一些物品的总重量为 $2lcm(1\sim n)$，那么一定存在这些物品的一个子集重量和为 $lcm(1\sim n)$。

然后直接维护十二个凸包，暴力用类似线段树的方式分治，然后暴力两两合并即可。

# [G - 战略游戏](https://www.luogu.com.cn/problem/P14991)

## 题目描述

有一张游戏地图上有 $n$ 个据点（编号为 $0$ 到 $n-1$ ），连接这些据点的是 $n-1$ 条单向道路，第 $i$ 条单向道路从编号为 $i$ 的据点出发，到达编号为 $p_i$ 的据点，最多通过 $k_i$ 次。有 $n$ 个玩家（编号同样为 $0$ 到 $n-1$ ）在给定的地图上玩战略游戏，初始时编号为 $i$ 的玩家控制编号为 $i$ 的据点。

我们定义一次进攻如下：

首先选择一个编号在 $1$ 到 $n-1$ 中的据点，假设这个据点编号为 $i$ ，然后控制第 $i$ 号据点的玩家通过第 $i$ 条道路一次，最后控制第 $p_i$ 号据点，原本控制第 $p_i$ 号据点的玩家则不再控制该据点。

你需要计算出：在若干次进攻（可以是 $0$ 次）之后，可能产生多少种不同的地图。两张地图不同当且仅当存在至少一个据点，在两张地图中控制它的玩家不同。

因为答案可能非常大，所以你只需要输出其对 $998244353$ 取模后的结果。

对于所有的测试数据，有 $1 \leq n \leq 3 \times 10^4$，$0 \leq p_i < i$，$1 \leq k_i \leq 15$。

## 初步观察
首先我们考虑每个点和每条边被颜色的覆盖情况，发现它们是不同的。
对于一条边，可能连续两次被染上相同的颜色（相邻相同），但是对于点来说，这是不可能的（相邻不同）。

## 基本定义
为了方便描述，我们来定义一些序列，这也是这个题目描述问题的关键：
- 点颜色序列：即一个点被染上过的颜色，但是不包括 $i$ 颜色本身，队尾就是点 $i$ 最后被染上的颜色。
- 上传颜色序列：从点 $i$ 走了 $i\rightarrow p_i$ 的颜色，仍然保证相邻不同。即点颜色序列选择在队头选择是否加入颜色 $i$，以及是否删除队尾。
- 边颜色序列：一条边被染过的颜色。即再上传颜色序列中选择其中一部分颜色复制若干次，并插入回原位置。

## dp 设计
由于我们的**点颜色序列**和**上传颜色序列**总是满足相邻颜色不同，所以我们只用记录序列的长度 $j$ 即可，设 $f_{i,j}$ 表示 $i\rightarrow p_i$ 的上传颜色序列，$g_{i,j}$ 表示 $i$ 的点颜色序列。设 $h$ 表示**边颜色序列**，具体如何设我们没有想好。

首先我们能够发现，每种根的上传颜色序列一定为空，如果有一个点的上传颜色序列不同，那么这就是不同的染色方案。

接下来，我们考虑用 dp 进行描述 $g\rightarrow f\rightarrow h\rightarrow g\rightarrow \dots$ 的循环，最后答案就是 $f_{0,0}$。

设每个儿子的点上传序列长度为 $a_i$，再转化为边颜色序列的时候拓展了 $b_i$ 次。
为了能贴合 $f,g$ 的状态，我们考虑同样只记录**边颜色序列**的长度，因为我们有要保证转移之后相邻颜色不相同，所以我们考虑容斥。
自然的，假设在后续转化为**点颜色序列**之后，仍然有 $t_i$ 个相邻。那么一个儿子的方案数即为：
$$
[a_i+b_i\le w_{to}]\prod_{t_i=0}^{b_i} (-1)^{t_i}{b_i\choose t_i}{a_i+b_i-1\choose b_i}
$$
最后还要任意归并，所以总方案数为：
$$
{(\sum (a_i+b_i-t_i))!\over \prod (a_i+b_i-t_i)!}\prod_{t_i=0}^{b_i} (-1)^{t_i}{b_i\choose t_i}{a_i+b_i-1\choose b_i} [a_i+b_i\le w_{to}]
$$
具体的，设 $h_{i,j,k}$ 表示 $i\rightarrow p_i$ 的边上，$\sum a_i + b_i = j,\sum a_i+b_i-t_i=k$ 的带容斥系数的方案数。
由于 $h$ 多设了一维，所以 $g$ 也要进行相应的调整，但这并不困难。

但是 $a_i$ 等于 $0$ 的时候组合数可能变成负数，需要特殊处理。

注意，这里第二维的 $j$ 是必要的，因为我们最后为了能够容斥掉不合法情况，要保证在转移到**点颜色序列**之后，他们的地位等价，也就是说，要在这个时候把 $t_i$ 个钦定相邻的位置解绑。

这样我们就可以简单做出 $f\rightarrow h$ 和 $h\rightarrow g$ 的转移。

对于 $g\rightarrow f$ 的转移，我们要把**点颜色序列**变成**上传颜色序列**，根据定义，我们讨论四种情况：
1. 加入 i，不删除队尾
2. 不加入 i，删除队尾
3. 不加入 i，不删除队尾
4. 加入 i，删除队尾

注意，在 $g_{0,0}$ 的时候没有 2,4 情况。

## 代码
```cpp
#include <bits/stdc++.h>

using namespace std;

#define ll long long

const int N = 30005;
const int mod = 998244353;

int n;
vector<pair<int, int> > e[N];
ll fac[40], inv[40], ifac[40], g[20][20], f[N][20], c[40][40], h[20][20], tmp[20][20];

void dfs(int u) {
    for (auto [v, w] : e[u]) dfs(v);
    memset(g, 0, sizeof(g));
    g[0][0] = 1;
    for (auto [v, w] : e[u]) {
        memset(h, 0, sizeof(h));
        h[0][0] = f[v][0];
        for (int a = 1; a <= w; a++) if (f[v][a]) {
            for (int b = 0; b + a <= w; b++) {
                for (int t = 0; t <= b; t++) {
                    (h[a + b][a + b - t] += (t & 1 ? mod - 1 : 1) * f[v][a] % mod * c[a + b - 1][b] % mod * c[b][t] % mod * ifac[a + b - t]) %= mod;
                }
            }
        }
        memcpy(tmp, g, sizeof(tmp));
        memset(g, 0, sizeof(g));
        for (int a = 0; a <= 16; a++) {
            for (int b = 0; b <= a; b++) if (tmp[a][b]) {
                for (int c = 0; c + a <= 16; c++) {
                    for (int d = 0; d <= c; d++) if (h[c][d]) {
                        (g[a + c][b + d] += tmp[a][b] * h[c][d]) %= mod;
                    }
                }
            }
        }
    }
    for (int i = 0; i <= 16; i++) {
        for (int j = 0; j <= 16; j++) if (g[i][j]) {
            (g[i][j] *= fac[j]) %= mod;
            if (i == 0) {
                (f[u][i] += g[i][j]) %= mod;
                (f[u][i + 1] += g[i][j]) %= mod;
            } else {
                (f[u][i - 1] += g[i][j]) %= mod;
                (f[u][i] += 2 * g[i][j]) %= mod;
                (f[u][i + 1] += g[i][j]) %= mod;
            }
        }
    }
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    for (int i = 0; i <= 30; i++) {
        c[i][0] = 1;
        for (int j = 1; j <= i; j++) {
            c[i][j] = c[i - 1][j - 1] + c[i - 1][j];
            if (c[i][j] >= mod) c[i][j] -= mod;
        }
    }
    fac[0] = fac[1] = inv[0] = inv[1] = ifac[0] = ifac[1] = 1;
    for (int i = 2; i <= 30; i++) {
        fac[i] = fac[i - 1] * i % mod;
        inv[i] = (mod - mod / i) * inv[mod % i] % mod;
        ifac[i] = ifac[i - 1] * inv[i] % mod;
    }
    cin >> n;
    for (int i = 1; i < n; i++) {
        int p, k;
        cin >> p >> k;
        e[p].push_back({i, k});
    }
    dfs(0);
    cout << f[0][0] << "\n";
    return 0;
}
```
