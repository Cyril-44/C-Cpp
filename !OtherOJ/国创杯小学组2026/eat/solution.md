# Solution

## 测试点 1～4
考虑用数组来记录这块巧克力。
$a_i$ 代表第 $i$ 段巧克力的美味度。
暴力枚举所有的选择方案，从中每次挑选最短的巧克力段吃掉，然后用同样的方法记录当前剩余的巧克力。
时间复杂度 $O(n^2)$。

我们可以发现，每次相当于删掉数组 $a$ 的一段前缀，或者后缀。
在这些前缀中，最左边满足契合度等于 $x$ 的位置最短。
在这些后缀中，最右边满足契合度等于 $x$ 的位置最短。
因此，最优解一定在这两种决策方案中，我们不妨直接考虑这两种方案中较短的那种。
时间复杂度 $O(n^2)$。

## 特殊性质 A
这个时候每次一定选择最左边的那块巧克力吃掉。
对于每次询问直接输出 `L 1`。

## 特殊性质 B
这是对正解的提示。
我们没有必要一定要找到最左边的和最右边的那块巧克力，只需要知道距离端点最近的那块位置在哪。现在题目保证了在 $10$ 块巧克力中一定可以找到，于是我们只需要扫描数组长度为 $10$ 的前缀和长度为 $10$ 的后缀，就一定能找到最优的吃巧克力方案。

## 特殊性质 C
既然每次吃巧克力的契合度只和这块巧克力自身有关，我们可以预处理出所有巧克力的契合度，并且给每一种契合度给一个新的编号，满足不同的契合度标号不同，可以按照契合度排序，从小到大标号 $1$ 到 $n$，这样我们可以对每一种契合度维护一个双向链表，这样就可以 $O(1)$ 找到最左边或者最右边可行的巧克力。

## 正解
**打破固有的循环枚举思路**。
假设现在剩余的巧克力重新编号后为 $1$ 到 $n$，从 $1$ 开始枚举 $i$ 的时候，我们可以同时计算 $i$ 和 $n - i + 1$， 的契合度是否等于 $x$，如果有一者等于 $x$ 那么我们就找到了最优的吃法，那么删除对应的巧克力段。
注：当删除一段前缀的时候，不要暴力移动数组，移动数组指针到当前最左边巧克力的位置即可。

## 代码
```cpp
#include <bits/stdc++.h>

using namespace std;

const int N = 1000005;

int n, q, A[N], *a = A;

int main() {
    scanf("%d%d", &n, &q);
    for (int i = 0; i < n; i++) scanf("%d", &a[i]);
    while (q--) {
        long long x;
        int t;
        scanf("%lld%d", &x, &t);
        for (int i = 1; ; i++) {
            if (1ll * a[i - 1] * a[(i - 1 + t) % n] == x) {
                printf("%d %d\n", 0, i - 1);
                a += i, n -= i;
                break;
            }
            if (1ll * a[n - i] * a[(n - i + t) % n] == x) {
                printf("%d %d\n", n - i, n - 1);
                n -= i;
                break;
            }
        }
    }
    return 0;
}
```
